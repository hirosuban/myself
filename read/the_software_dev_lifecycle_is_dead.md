# TR;DR
- ソフトウェア開発ライフサイクルは死んだ

## 記憶に残ったこと
- Cursor以降にキャリア始めたエンジニアは、DevOpsとかSREとか知らない
  - スプリント計画に出たこともない、ストーリーポイントを見積もったこともない
  - PRレビューを待ったこともない
  - ただ、欲しいものを説明して、エージェントがコード書いて、かくにんして、リリースするだけ
- 要件定義は流動的になった
  - 作ることがコストだったので、要件は事前に凍結された
  - エージェントが数分で機能生成できるので、事前に細部詰めなくて良い
  - JIRA的なプロジェクトツールは、ただの文脈保存庫
- ⭐️設計は依然大事。やり方が変わった
  - 設計は人間が命じるものではない
  - エージェントと対話して、良い提案をもらう
- テスト:同時生成
  - TDDは方法論ではなく、デフォルトの操作
  - QAという独立した段階は消える
- ⭐️PRレビュー：手放せ。過去の異物
  - エージェントが1日に数回コード生成するので、PRレビューは追いつかない
- デプロイ:　常時実行
  - リリースパイプラインまで自動で作ってくれるから
- ⭐️監視: 最後に残る工程。
  - むしろ基盤になる
  - よくあるダッシュボードでは足りない。人間の認知を超えるから
  - フィードバック中心。ここもテレメトリをAIに渡して、自己修復してもらう
- 残るのは？
  - コンテキストエンジニアリング
  - それでも業界の多くは、誰もみないダッシュボードを設定し続けている

## 感想
- QAという独立した段階は消えるけど、QA自体は残るよね〜（確認、監視作業という意味でも）
